Column Information



Classification:

Defines how a table is used.

ValueDescription
StaticTable data is not modified, inserted or deleted often.
DynamicTable data is modified, inserted or deleted often.


Maintenance:

A bit mask value indicating how table data is cleaned up.

Values:
ValueBit ValueDescription
Manual1Data can be cleaned up manually through Actions/Purge screen in the Portal.
Scheduled2Data is automatically cleaned up when the Database Maintenance process is executed.
Parent Table4Data is not cleaned up directly but indirectly through a parent table (Cascading).
Repository8Identifies the repository tables. These are specialized cleanup tables.
Not Allowed16No cleanup of data is done.


Publication:

Defines whether a table is included in a publication and which publication(s) it is.

Values:
  • LaborWorkx_Transactional
  • LaborWorkx_Merge_ReadOnly
  • LaborWorkx_Merge_Updatable


Historical Data Transfer Method:

Defines how data is transferred to the Historical Database.

Values:
ValueDescription
CopyA specific set of data in the table is copied to the historical database.
MoveA specific set of data in the table is moved to the historical database.
AppendThe data in the table is transferred to the historical database based on the last date transferred (used by audit tables).
SnapshotThe entire table is copied to the historical database.
AppendNonPayPeriodThe data in the table is transferred to the historical database based on the last date transferred and a rolling date window.
MoveNonPayPeriodThe data in the table is transferred to the historical database based on the last date transferred and a rolling date window.


Audit Table:

Defines the audit table used for auditing.

CleanUpScriptAction:

Defines how a table is cleaned before shipment.

Values:
ValueDescription
NoneNothing is done.
TrunReseedRemove all data and reseed ID value if present.
TruncateRemove all data.
CoreDatacore data script is used to populate data.
